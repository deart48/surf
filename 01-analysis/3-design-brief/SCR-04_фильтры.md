# SCR-04 · Фильтры списка классов

**Метаданные**

- **Роль:** Клиент (авторизован).
- **Приоритет:** Must.
- **Связанные требования:** FR-4, UC-1 (шаг 3, A1 — больший период, A2 — сброс к дефолту), US-3, NFR-2, NFR-6, NFR-10.
- **Точки входа:** кнопка/иконка фильтров на списке классов (SCR-03).
- **Точки выхода:** возврат на список классов (SCR-03) после «Применить» (с обновлённой выдачей и чипами), после «Сбросить» (к дефолтной выдаче), или по закрытию панели без изменений.

## 1. Роль экрана и задача пользователя

Это инструмент «сузить каталог под себя», а не самостоятельный раздел. Задача клиента: быстро отсечь лишнее и получить только те классы, которые ему подходят — «покажи опытные классы со свободными местами у Марко на следующие две недели». Панель должна помогать принять решение здесь же (видно, сколько классов найдётся), а не гонять человека туда-обратно.

Фильтры **read-only по источникам**: список шефов и типы программ приходят из бэкенда; пользователь только выбирает из готовых значений, ничего не заводит сам.

## 2. Контекст, точки входа и выхода

Панель вызывается со списка классов (SCR-03) и работает поверх него. По результату применения пользователь возвращается на SCR-03 с суженной выдачей; активные условия отражаются там **чипами**. Если фильтры дали пустой результат — на SCR-03 показывается состояние **E1** («под фильтры ничего не нашлось»), а не E2.

**Форм-фактор (предлагаем оба):**

- **Desktop web — боковая панель** (лево или право), которая может оставаться открытой рядом со списком; изменения видно сразу на списке.
- **Mobile web — модальное окно / bottom-sheet** поверх списка на всю или большую высоту экрана, с фоновым затемнением; список под ним не прокручивается, пока панель открыта.

## 3. Информационная архитектура и контент

Ровно **четыре** группы фильтров (и ни одной сверх требований):

1. **Дата / период старта.** Выбор дня или диапазона. Ключевая возможность — задать **период шире 7 дней** (UC-1 A1). Дефолт (когда фильтр не трогали) — ближайшие 7 дней. Уместны быстрые пресеты («Ближайшие 7 дней», «Эта неделя», «Следующие 2 недели») плюс произвольный диапазон.
2. **Тип программы.** Новичковый / опытный. Множественный выбор (можно оба = без ограничения по типу).
3. **Наличие свободных мест.** Переключатель «Только со свободными местами» (вкл/выкл). По умолчанию выкл — показываем все, включая заполненные (они видны на SCR-03 как «Мест нет»).
4. **Шеф.** Выбор одного или нескольких шефов из **списка, полученного от бэкенда**. Если шефов много — с поиском по имени.

Дополнительно: **счётчик найденного** («Показать 12 классов») и управляющие кнопки (§4).

## 4. Ключевые блоки и элементы интерфейса

- **Заголовок панели** «Фильтры» + кнопка закрытия (крестик / «назад»).
- **Группа «Дата/период»:** пресеты периодов + выбор произвольного диапазона (от–до). Понятно показывает текущий выбор и то, что можно выйти за пределы недели.
- **Группа «Тип программы»:** переключаемые опции «Новичковый», «Опытный» (чипы/тумблеры множественного выбора).
- **Группа «Свободные места»:** одиночный переключатель «Только со свободными местами».
- **Группа «Шеф»:** список выбираемых шефов (чекбоксы), с поиском при длинном списке; каждый — имя (при наличии аватар).
- **Счётчик результата:** динамически показывает, сколько классов найдётся при текущем наборе условий (обновляется по мере выбора).
- **Кнопки действий (закреплены внизу/в футере панели):**
  - **«Применить»** — применяет фильтры, закрывает панель (на mobile) или обновляет выдачу (на desktop), возвращает на SCR-03 с чипами.
  - **«Сбросить»** — очищает все условия и возвращает выдачу к дефолту: **ближайшие 7 дней** (UC-1 A2). Кнопка неактивна, если фильтры уже в дефолте.

## 5. Сценарии и взаимодействия

- **Задать фильтры (UC-1 шаг 3).** Клиент открывает панель → выбирает условия → счётчик обновляет число найденных → «Применить» → SCR-03 с суженной выдачей и чипами.
- **Больший период (UC-1 A1).** Клиент задаёт диапазон шире 7 дней (пресет «2 недели» или произвольный от–до) → «Применить» → SCR-03 показывает слоты за выбранный период (список длиннее, сгруппирован по дням).
- **Сброс к дефолту (UC-1 A2).** «Сбросить» → все условия очищены, выдача = ближайшие 7 дней; чипы на SCR-03 исчезают.
- **Живой предпросмотр количества.** По мере изменения условий счётчик найденного пересчитывается (NFR-6 — быстро; допустим лёгкий дебаунс/индикатор пересчёта).
- **Desktop — открытая боковая панель.** Изменение фильтра может обновлять список сразу, без обязательного «Применить»; тем не менее «Применить»/«Сбросить» остаются явными для предсказуемости.
- **Закрытие без применения.** Крестик/тап по затемнению (mobile) закрывает панель, не меняя текущую выдачу.
- **Пустой результат.** Если по выбранным условиям ничего не найдётся, счётчик показывает «0», а после «Применить» на SCR-03 — состояние E1 с подсказкой изменить/сбросить фильтры.

## 6. Состояния экрана

- **Загрузка справочников.** Списки для фильтров (шефы, типы программ) приходят из API — пока грузятся, показываем скелетоны опций; кнопка «Применить» доступна, дефолтные группы (дата, наличие мест) работают сразу.
- **Пусто (нет данных для фильтра).** Если бэкенд не вернул шефов/типов — соответствующая группа скрывается или помечается «недоступно», остальные фильтры продолжают работать.
- **Пустой предпросмотр.** Счётчик = «0 классов»; можно заранее подсказать «Под эти условия классов нет — попробуйте расширить период». Итоговое пустое состояние (E1) живёт на SCR-03.
- **Ошибка загрузки справочников.** Понятное сообщение + «Повторить»; фильтры, не зависящие от справочников (дата, наличие мест), остаются рабочими.
- **Успех.** Условия применены, выдача обновлена, чипы отражают выбор на SCR-03.
- **Нет доступа.** Доступно только авторизованному клиенту; отдельного состояния внутри панели не требуется.

## 7. Крайние случаи, ограничения и валидация

- **Диапазон дат.** Дата «до» не раньше «от»; нижняя граница — не в прошлом (каталог предстоящих классов). Некорректный диапазон блокирует «Применить» с понятной подсказкой.
- **Очень широкий период.** Разрешён (A1), но стоит помнить о длинной выдаче на SCR-03 (группировка, подгрузка).
- **Пересчёт — источник истины бэкенд (NFR-10, R-004).** Счётчик найденного и итоговый список считает бэкенд; UI не фильтрует/не пересчитывает места локально помимо запроса.
- **Сочетание фильтров.** Все четыре группы комбинируются логическим «И» между группами; внутри группы (типы, шефы) — «ИЛИ». Это стоит отразить в поведении счётчика.
- **Никаких лишних фильтров.** Только дата/период, тип программы, наличие мест, шеф. Не добавлять фильтры по цене, длительности, адресу и т.п. — их нет в требованиях.
- **Синхронизация с чипами.** Снятие чипа на SCR-03 эквивалентно снятию условия в панели; при повторном открытии панель отражает актуальное состояние.

## 8. Тон, иерархия и визуальные акценты

Панель — спокойный, аккуратный инструмент, не перегруженный. Самая частая и важная группа — **дата/период** (и возможность выйти за 7 дней) — вверху и заметна. Тип программы — второй по важности. «Только со свободными местами» — простой понятный тумблер. Счётчик найденного — заметный, он снимает неуверенность («стоит ли жать Применить»). Кнопки «Применить»/«Сбросить» всегда на виду (закреплённый футер), «Применить» — основное действие. Тон дружелюбный, без канцелярита.

## 9. Доступность и web-специфика

- **Адаптивность.** Desktop — боковая панель (может сосуществовать со списком); mobile — bottom-sheet/модалка с затемнением и закреплённым футером кнопок.
- **Фокус и клавиатура.** При открытии фокус переходит внутрь панели; **фокус-ловушка** внутри модалки на mobile; Esc закрывает панель; Tab обходит группы и кнопки в логичном порядке; видимый фокус.
- **Возврат фокуса.** После закрытия фокус возвращается на кнопку фильтров на SCR-03.
- **Скринридеры.** Панель объявляется как диалог с заголовком «Фильтры»; группы имеют подписи; переключатели и чекбоксы озвучивают состояние; счётчик найденного — «живая» область (aria-live), чтобы озвучивать пересчёт.
- **Не только цвет.** Выбранные опции обозначены не только цветом (галочка/заливка/подпись).
- **Отзывчивость (NFR-6).** Пересчёт счётчика и применение — быстрые; на слабой сети — деликатный индикатор пересчёта, без блокировки.

## 10. Критерии готовности дизайна (чек-лист)

- [ ] Ровно четыре группы фильтров: дата/период, тип программы, наличие мест, шеф — без лишних.
- [ ] Дата/период позволяет задать диапазон **шире 7 дней** (A1) + быстрые пресеты.
- [ ] Переключатель «Только со свободными местами».
- [ ] Список шефов и типы программ — из бэкенда (read-only), с поиском при длинном списке шефов.
- [ ] Счётчик найденных классов, обновляемый по мере выбора.
- [ ] Кнопки «Применить» и «Сбросить»; «Сбросить» возвращает к дефолту (ближайшие 7 дней, A2).
- [ ] Обе раскладки: desktop боковая панель и mobile bottom-sheet/модалка.
- [ ] Связь с SCR-03: применённые фильтры → чипы; пустой результат → E1 на SCR-03.
- [ ] Состояния: загрузка справочников, ошибка загрузки, пустой предпросмотр.
- [ ] Доступность: фокус-ловушка, Esc, возврат фокуса, озвучивание диалога и счётчика, статусы не только цветом.

## 11. Открытые вопросы на согласование

1. Desktop: применять фильтры сразу при изменении (живая боковая панель) или только по «Применить»?
2. Набор пресетов периода: какие именно («7 дней», «эта неделя», «2 недели», произвольный)?
3. Тип программы и шеф — множественный выбор (как заложено) или одиночный?
4. Есть ли верхняя граница глубины периода (например, максимум горизонта расписания от бэкенда)?
5. Нужен ли на самих кнопках счётчик («Показать 12 классов» прямо в «Применить») или отдельной строкой?
6. Как обходимся, если у части слотов шеф не назначен, — нужен ли пункт «Шеф не указан» в фильтре?
