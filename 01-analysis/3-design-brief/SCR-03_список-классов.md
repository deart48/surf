# SCR-03 · Список классов (каталог слотов)

**Метаданные**

- **Роль:** Клиент (авторизован).
- **Приоритет:** Must.
- **Связанные требования:** FR-3, FR-4 (вход в фильтры), FR-5 (краткая карточка), UC-1 (осн. поток, A1, A2, E1, E2), US-2, US-3, NFR-2, NFR-6.
- **Точки входа:** первый экран после успешного входа (SCR-02 → сюда); переход из нижней/боковой навигации по вкладке «Классы»; возврат «назад» с карточки класса (SCR-05); возврат после отмены/применения фильтров (SCR-04); переход с экрана «Запись создана» (SCR-07) по кнопке «К списку классов».
- **Точки выхода:** карточка класса (SCR-05) по тапу на слот; панель фильтров (SCR-04) по кнопке фильтров; другие разделы через навигацию («Мои брони» SCR-08, «Профиль» SCR-10).

## 1. Роль экрана и задача пользователя

Это витрина студии и главный экран приложения. Задача клиента здесь простая и живая: «покажи, на какие классы я могу попасть в ближайшие дни, и дай быстро выбрать подходящий». Человек пришёл готовить, а не разбираться в интерфейсе — он должен с первого взгляда понять, что доступно, когда, с каким шефом и по какой цене, и одним движением провалиться в детали.

Экран отвечает на вопросы: «Что готовят на этой неделе?», «Есть ли ещё места?», «Сколько стоит?», «Кто ведёт?». Всё остальное (запись, гости, экипировка) — уже на следующих экранах. Экран **только для чтения**: клиент ничего не создаёт и не редактирует, расписание приходит из существующей инфраструктуры.

## 2. Контекст, точки входа и выхода

По умолчанию, без каких-либо действий пользователя, экран показывает **предстоящие слоты на ближайшие 7 дней** от текущего момента, отсортированные по времени старта (ближайший — сверху). Это совпадает с горизонтом планирования студии («расписание на неделю вперёд»).

Клиент попадает сюда сразу после входа и возвращается многократно за сессию. Отсюда два основных пути вглубь: открыть **карточку класса** (тап по слоту) и открыть **фильтры**. После применения фильтров пользователь возвращается на этот же экран с уже суженной выдачей и видимыми активными фильтрами.

## 3. Информационная архитектура и контент

Приоритет контента сверху вниз:

1. **Заголовок раздела и краткий контекст выдачи** — что именно показано: «Ближайшие классы» / диапазон дат, если применён фильтр периода. Помогает не потеряться, особенно когда фильтр расширил горизонт.
2. **Строка активных фильтров** — чипы применённых условий и быстрый сброс (см. §4). Показывается только когда фильтры заданы.
3. **Список слотов**, при необходимости **сгруппированный по дням** (заголовок-дата над группой: «Сегодня», «Завтра», «Сб, 11 июля»).
4. Каждый элемент списка — **краткая карточка слота** (не путать с полной карточкой SCR-05). В ней ровно то, что нужно для выбора:
   - дата и **время старта** (класс идёт ~3 часа — можно показать «10:00, ~3 ч» или интервал);
   - **программа / меню** и её **тип**: новичковый / опытный (тип — заметная метка, это ключевой критерий выбора);
   - **шеф** (имя, при наличии — небольшой аватар);
   - **цена** за место;
   - **мест: всего и свободно** (например, «свободно 4 из 24» или компактно «4 места»).

Чего здесь **нет** (это уровень карточки SCR-05, не списка): адрес студии, подробное описание меню, детальная доступность прокатных комплектов, форма записи.

## 4. Ключевые блоки и элементы интерфейса

- **Шапка списка.** Название раздела, контекст выдачи (период), кнопка/иконка входа в **фильтры** (SCR-04). Кнопка фильтров показывает, что фильтры активны (бейдж с числом применённых условий).
- **Чипы активных фильтров.** Горизонтальный ряд чипов: «Опытный», «Со свободными местами», «Шеф: Марко», «12–20 июля». Каждый чип снимается по тапу (крестик). Рядом — действие **«Сбросить всё»**, возвращающее выдачу к дефолту (ближайшие 7 дней). Соответствует UC-1 A2.
- **Разделители-даты (группировка по дням).** Заголовок дня перед группой слотов; помогает ориентироваться в длинной выдаче и при расширенном периоде.
- **Краткая карточка слота.** Кликабельная целиком. Содержит контент из §3. Визуально выделяет тип программы и наличие мест. Карточка **слота без свободных мест остаётся видимой**, но помечена «Мест нет» и визуально приглушена (см. §6, §7).
- **Индикатор загрузки** — скелетоны карточек (см. §6).
- **Пустые и ошибочные состояния** — крупный блок в центре области списка (см. §6).

## 5. Сценарии и взаимодействия

- **Просмотр по умолчанию (UC-1 осн. поток).** Клиент открывает экран → видит слоты на 7 дней по времени старта → выбирает подходящий → тап по карточке → SCR-05.
- **Открыть фильтры (FR-4).** Тап по кнопке фильтров → SCR-04. После «Применить» — возврат сюда с обновлённой выдачей и чипами.
- **Снять один фильтр.** Тап по крестику на чипе → выдача мгновенно пересчитывается (запрос к бэкенду), контекст-заголовок обновляется.
- **Сбросить все фильтры (UC-1 A2).** «Сбросить всё» → возврат к дефолтной выдаче (ближайшие 7 дней).
- **Расширенный период (UC-1 A1).** Если через фильтр задан период шире 7 дней — список может стать длинным; группировка по дням и понятный заголовок периода обязательны.
- **Тап по слоту без мест.** Карточка кликабельна и ведёт на SCR-05 (посмотреть детали можно всегда), но запись недоступна — это честно сообщается уже в списке меткой «Мест нет».
- **Возврат на экран.** При возврате с карточки/фильтров сохраняются заданные фильтры и, по возможности, позиция прокрутки.

## 6. Состояния экрана

- **Загрузка.** Скелетоны нескольких карточек (форма будущих слотов), без «крутилки» на весь экран — чтобы структура была понятна заранее и ощущение отклика было мгновенным (важно для NFR-6, пиковые выходные).
- **Пусто — E2 «Пока нет доступных классов» (нет расписания вовсе).** Показывается, когда **фильтры не заданы**, а в горизонте выдачи нет ни одного предстоящего слота. Тёплое, спокойное сообщение: расписание ещё не опубликовано / на ближайшие дни классов нет; предложить заглянуть позже. **Кнопки «сбросить фильтры» здесь быть не должно** — сбрасывать нечего. Уместно мягкое действие «Обновить».
- **Пусто — E1 «Под фильтры ничего не нашлось» (результатов нет из-за фильтров).** Показывается, когда **фильтры заданы**, но выдача пустая. Сообщение прямо говорит: под выбранные условия классов нет, и подсказывает выход — **изменить или сбросить фильтры**. Обязательны действия «Сбросить фильтры» и «Изменить фильтры» (открыть SCR-04). Активные чипы остаются видны, чтобы человек понимал, что именно сузило выдачу.
  > E1 и E2 — **разные** состояния: E2 = «студия ничего не запланировала», E1 = «запланировано, но не под твой фильтр». Формулировки и доступные действия должны их чётко различать.
- **Ошибка сети.** Список не загрузился: понятное сообщение «Не удалось загрузить классы» + кнопка **«Повторить»**. Не терять уже заданные фильтры при повторе.
- **Частичный/фоновый рефреш.** При обновлении выдачи (снятие чипа, возврат на экран) — деликатный индикатор, без полного сброса в скелетон, если данные уже были.
- **Успех.** Собственно наполненный список слотов (основное состояние).
- **Нет доступа.** Экран доступен только авторизованному клиенту; при истёкшей сессии — переход на вход (SCR-01). Отдельного «нет доступа» внутри списка не требуется.

## 7. Крайние случаи, ограничения и валидация

- **Слот без свободных мест.** Карточка видна, помечена «Мест нет», приглушена, но тап по ней открывает SCR-05 (детали доступны). Запись при этом недоступна — на SCR-05/SCR-06.
- **Слот, отменённый студией.** Отменённые слоты в каталоге предстоящих классов не предлагаются к записи (повторная запись запрещена, R-008 / FR-18). Если такой слот всё же отображается, он явно помечен и не ведёт к записи. (Основная работа с «Отменён студией» — в «Моих бронированиях», SCR-08/09.)
- **Граница «ближайшие 7 дней».** Отсчёт от текущего момента, не от начала суток; слоты, которые вот-вот стартуют, но ещё не начались, остаются в списке до старта. «Прошедшесть» определяется по `start_at`, а не по хранимому статусу.
- **Свободные места — источник истины бэкенд (NFR-4, R-004).** Число мест и «свободно» приходят из API; UI не пересчитывает лимиты. Возможно расхождение между просмотром и записью (места разобрали) — на этом экране отражается при рефреше; окончательная защита — на записи (SCR-06).
- **Длинная выдача (A1).** При широком периоде — постраничная подгрузка/бесконечная прокрутка; группировка по дням обязательна.
- **Разные часовые пояса/локаль.** Время — в часовом поясе студии; форматирование даты — по локали, без двусмысленности («Сб, 11 июля, 10:00»).
- Экран **read-only**: никаких действий создания/редактирования расписания, никаких кнопок «добавить класс».

## 8. Тон, иерархия и визуальные акценты

Тёплый, «вкусный», гостеприимный образ студии в лофте. Главные акценты в карточке — **время старта**, **тип программы** (новичковый/опытный) и **наличие мест**: именно по ним человек фильтрует взглядом. Цена — заметна, но вторична к «когда и что». Тип программы стоит различать не только цветом (доступность, §9). «Мест нет» — спокойный приглушённый акцент, без тревожного красного алерта; это норма, а не ошибка. Пустые состояния — дружелюбные, без канцелярита; E2 особенно тёплое («Скоро добавим новые классы»).

## 9. Доступность и web-специфика

- **Адаптивность.** Desktop web: список в одну колонку с комфортной шириной карточек либо сетка 2–3 колонки на широких экранах; фильтры — вход в боковую панель. Mobile web: одна колонка, карточки на всю ширину, вход в фильтры — заметная кнопка, панель открывается как bottom-sheet/модалка (см. SCR-04).
- **Клавиатура.** Карточки и чипы достижимы по Tab, открываются Enter/Space; порядок фокуса сверху вниз, слева направо; видимый фокус-ринг.
- **Скринридеры.** Каждая карточка озвучивается связно: «11 июля, 10:00, программа X, опытный, шеф Марко, 2500 ₽, свободно 4 места». Метка «Мест нет» — часть доступного имени, а не только цвет.
- **Не только цвет.** Тип программы и статус «Мест нет» дублируются текстом/иконкой.
- **Отзывчивость (NFR-6).** Мгновенные скелетоны, лёгкий фоновый рефреш, отсутствие блокирующих оверлеев — критично в пиковые выходные, когда одновременно заходит максимум клиентов.

## 10. Критерии готовности дизайна (чек-лист)

- [ ] Дефолтная выдача: ближайшие 7 дней, сортировка по времени старта, опциональная группировка по дням.
- [ ] Краткая карточка слота содержит: дату/время старта, программу и тип, шефа, цену, всего/свободно мест — и ничего лишнего из уровня SCR-05.
- [ ] Вход в фильтры + бейдж активных фильтров; строка чипов с посниманием и «Сбросить всё».
- [ ] Спроектированы **оба** разных пустых состояния: E2 (нет расписания, без «сбросить фильтры») и E1 (нет под фильтры, с «изменить/сбросить фильтры»).
- [ ] Состояние загрузки — скелетоны карточек.
- [ ] Состояние ошибки сети с «Повторить», без потери фильтров.
- [ ] Слот без мест: карточка видна, помечена «Мест нет», ведёт на SCR-05.
- [ ] Desktop и mobile web раскладки; вход в фильтры для обоих.
- [ ] Доступность: клавиатура, фокус, озвучивание карточки, статусы не только цветом.
- [ ] Никаких элементов создания/редактирования расписания (read-only).

## 11. Открытые вопросы на согласование

1. Группировка по дням — включаем всегда или только при периоде > 7 дней?
2. Как показываем «свободно мест» в краткой карточке: точное число, «свободно N», или порог «мало мест» при небольшом остатке?
3. Нужен ли в списке хотя бы намёк на доступность прокатной экипировки, или это оставляем полностью на SCR-05?
4. Длинная выдача (A1): бесконечная прокрутка или «Показать ещё»/страницы?
5. Показывать ли в каталоге предстоящих слоты, отменённые студией, — скрывать полностью или помечать неактивными?
6. Нужен ли ручной pull-to-refresh / кнопка обновления, помимо автоматического рефреша?
