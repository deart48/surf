## TC-AUTH — Авторизация (SCR-01/02, LOGIC-001/002)

| ID | Название | Приоритет | Предусловия | Шаги | Ожидаемый результат |
| :-- | :-- | :--: | :-- | :-- | :-- |
| TC-AUTH-01 | Успешный вход returning user | P0 | Номер уже зарегистрирован в БД | 1. Открыть `/login`<br>2. Ввести валидный номер +7…<br>3. «Получить код»<br>4. Ввести OTP (не `0000`)<br>5. «Подтвердить» | Redirect на `/classes`; access/refresh токены сохранены |
| TC-AUTH-02 | Регистрация нового клиента | P0 | Номер отсутствует в БД | 1–4 как TC-AUTH-01<br>5. На форме «Как к вам обращаться?» ввести имя<br>6. «Продолжить» | `PATCH /profile` с именем; redirect на `/classes` |
| TC-AUTH-03 | Неверный OTP | P0 | На SCR-02 после отправки кода | Ввести `0000` → «Подтвердить» | Сообщение об ошибке; поля OTP очищены; фокус на первой цифре |
| TC-AUTH-04 | Невалидный телефон | P1 | `/login` | Ввести короткий/битый номер → submit | «Проверьте номер — кажется, не хватает цифр»; кнопка disabled до валидного номера |
| TC-AUTH-05 | Route guard неавторизованного | P0 | Токены отсутствуют | Открыть `/classes` или `/bookings` | Redirect на `/login` |
| TC-AUTH-06 | Silent refresh сессии | P1 | Access истёк, refresh валиден | Выполнить защищённый GET (напр. `/profile`) | Авто `POST /auth/refresh`; запрос повторён; пользователь остаётся в АЗ |
| TC-AUTH-07 | Таймер повторной отправки OTP | P1 | Код успешно отправлен | На SCR-02 дождаться блокировки кнопки resend | Кнопка disabled N сек, где N = `resend_after_seconds` из `POST /auth/request-code` |
| TC-AUTH-08 | Paste OTP из буфера | P2 | SCR-02 | Вставить 4 цифры в первое поле | Все поля заполнены; фокус на последней цифре |
| TC-AUTH-09 | Logout | P1 | Авторизован | Профиль → «Выйти» → подтвердить | `POST /auth/logout`; токены очищены; redirect `/login` |

**Трассировка:** FR-1, UC-1, LOGIC-001, LOGIC-002

| TC-CAT-02 | Фильтр «только доступные» | P1 | Фильтры → `only_available` → Применить | Только слоты с `free_seats > 0` |
| TC-CAT-03 | Фильтр по типу программы и шефу | P1 | Выбрать `program_type[]` и `chef_id[]` → Применить | Список соответствует фильтрам |

| TC-CAT-06 | Карточка класса SCR-05 | P0 | Клик по слоту в списке | Все параметры слота; CTA «Записаться» active/disabled по `status` и `free_seats` |
| TC-CAT-07 | Отменённый слот в каталоге | P1 | Слот `status=cancelled` в выдаче | Метка «Класс отменён»; CTA «Запись недоступна» |
| TC-CAT-08 | Слот без мест | P1 | `free_seats = 0` | «Мест нет»; CTA disabled |

## TC-BOOK — Бронирование (SCR-06/07, LOGIC-003/004)

| ID | Название | Приоритет | Шаги | Ожидаемый результат |
| :-- | :-- | :--: | :-- | :-- |
| TC-BOOK-01 | Успешная запись | P0 | SCR-05 → «Записаться» → 1 место, 0 прокат → «Подтвердить» | 201; redirect `/bookings/{id}/success` |
| TC-BOOK-02 | Live-пересчёт цены | P0 | Менять `seats_count` / `rental_count` | Итого = `price × seats + rental_price × rental` без перезагрузки |
| TC-BOOK-03 | Лимит мест 1..min(free_seats, 6) | P0 | Слот с `free_seats = 2` | Counter мест: max = 2 |
| TC-BOOK-04 | Лимит проката 0..min(seats, free_rental_sets) | P0 | Увеличить места при ограниченном прокате | Counter проката ограничен корректно |
| TC-BOOK-05 | Ошибка 409 slot_full | P1 | Забронировать больше свободных мест (или параллельно занять места) | Сообщение об ошибке; актуальные `available_seats` в UI |
| TC-BOOK-06 | Ошибка 409 rental_unavailable | P1 | `rental_count` > доступного проката | Сообщение; актуальные `available_rental_sets` |
| TC-BOOK-07 | Ошибка 410 slot_cancelled | P0 | Submit на отменённом слоте | «Класс отменён студией. Повторная запись недоступна.» |
| TC-BOOK-08 | Идемпотентность Idempotency-Key | P0 | Двойной клик «Подтвердить» или retry после сетевой ошибки | Одна бронь; повтор с тем же ключом → 201 с тем же `id` |
| TC-BOOK-09 | Поле аллергий | P2 | Ввести текст до 500 символов | Значение сохранено в брони |
| TC-BOOK-10 | Экран успеха SCR-07 | P1 | После 201 | Сводка брони, цена, офлайн-оплата, push-блок, ссылки «Мои бронирования» / «К классам» |

**Трассировка:** FR-6, FR-7, UC-2, LOGIC-003, LOGIC-004

## TC-MY — Мои бронирования и отмена (SCR-08/09, LOGIC-005/006)

| ID | Название | Приоритет | Шаги | Ожидаемый результат |
| :-- | :-- | :--: | :-- | :-- |
| TC-MY-01 | Вкладка «Предстоящие» | P0 | Есть active-брони с будущим `start_at` | Только предстоящие active-брони |
| TC-MY-02 | Вкладка «История» | P1 | Есть прошлые / отменённые брони | Брони с прошедшим `start_at` или `status ≠ active` |
| TC-MY-03 | Детали брони | P0 | Клик по карточке | SCR-09: программа, дата, шеф, места, прокат, цена, статус |
| TC-MY-04 | Ранняя отмена (≥ 24 ч) | P0 | Active-бронь, до старта ≥ 24 ч | Модалка «ранняя отмена» → `status = cancelled`; места освобождены |
| TC-MY-05 | Поздняя отмена (< 24 ч) | P0 | Active-бронь, до старта < 24 ч | Модалка «поздняя отмена» → `status = late_cancel`; места не освобождены |
| TC-MY-06 | studio_cancelled | P1 | Бронь на слот, отменённый студией | Бейдж + `cancel_reason`; CTA отмены отсутствует |
| TC-MY-07 | Модалка отмены — a11y | P2 | Открыть модалку | Esc закрывает; фокус-ловушка; возврат фокуса после закрытия |

**Трассировка:** FR-8, FR-9, UC-3, UC-4, LOGIC-005, LOGIC-006
